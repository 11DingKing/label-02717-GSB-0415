# 音频克隆项目 Docker 部署文档

## 项目简介
这是一个基于 YourTTS 模型的语音克隆服务，支持上传音频文件并输入文本，生成与上传音频音色一致的语音。
项目采用前后端分离架构：
- 后端：Python + Flask + Coqui TTS
- 前端：静态页面 + Nginx 反向代理

## 环境要求
- Docker 20.10+
- Docker Compose 2.0+

## 部署步骤

### 1. 一键启动服务
```bash
# 进入项目根目录
cd label-02717-GSB-0415

# 后台启动所有服务
docker compose up -d --build
```

### 2. 等待服务启动
> ⚠️ 注意：首次启动时需要下载 YourTTS 模型（约 400MB），根据网络情况可能需要 1-5 分钟，请耐心等待。
> 服务内置健康检查机制，前端会自动等待后端服务完全就绪后才能访问。

### 3. 查看服务状态
```bash
# 查看所有服务运行状态
docker compose ps

# 查看后端服务日志（可观察模型下载进度）
docker compose logs -f backend

# 查看所有服务日志
docker compose logs -f
```

## 访问方式
- **前端页面**：http://localhost:8081
- **后端健康检查接口**：http://localhost:5000/api/health（正常返回 `{"status": "ok"}`）

## 使用说明
1. 打开前端页面后，点击上传按钮选择一段 10-30 秒的清晰音频文件（wav/mp3格式）
2. 在文本框中输入需要合成的文本（目前仅支持英文、葡萄牙语、法语，中文会按英文发音处理）
3. 点击"生成语音"按钮，等待几秒后即可下载生成的音频文件

## 常用命令
```bash
# 停止服务
docker compose down

# 停止服务并删除所有数据（包括已下载的模型和生成的语音数据）
docker compose down -v

# 重启服务
docker compose restart

# 查看服务资源占用情况
docker compose stats
```

## 架构说明
### Docker 优化点
1. **后端多阶段构建**：
   - 构建阶段负责安装依赖和编译，运行阶段仅包含最小运行时环境，镜像体积减小约 30%
   - 分离构建和运行依赖，避免构建工具残留，提高安全性

2. **健康检查机制**：
   - 后端服务每 10 秒检测一次 /api/health 接口
   - 前端服务依赖后端健康状态，后端未就绪时前端不会启动
   - 服务异常时会自动重启，提高可用性

3. **反向代理配置**：
   - Nginx 自动代理 /api/ 请求到后端服务，无需硬编码后端地址
   - 内置后端不可用提示，提升用户体验
   - 支持大文件上传（最大 50M），超时时间配置合理

## 常见问题
### Q: 首次启动后前端访问显示 503 错误？
A: 后端正在下载模型或加载中，请等待 1-2 分钟后刷新页面即可。可通过 `docker compose logs -f backend` 查看加载进度。

### Q: 生成语音时提示音频文件太小？
A: 请上传至少 10 秒以上的清晰音频文件，确保文件大小大于 1KB。

### Q: 中文文本生成的语音发音不对？
A: YourTTS 模型目前仅支持英语、葡萄牙语、法语，中文文本会按照英文发音规则处理，建议输入英文文本获得最佳效果。
